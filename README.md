# scBC
scBC —— a single-cell transcriptome Bayesian biClustering framework

## simulation procedure

> <u>In this section, we will briefly go through the simulation procedure as described in the paper.</u>

Before we start, make sure your folder contains source files as follows:

- `simulation.R`
- `permutaion.R`
- `match.R`
- `GBC_EM.R`
- `DWL.R`
- `BIC_tuning.R`
- `plaid_Xmotif.R`
- `analyse.R`

and ensure you are ready to use the python package `scvi-tools` in `R`, you can look up to the [installation guide](https://docs.scvi-tools.org/en/stable/installation.html) for reference. The must-have R packages are: `reticulate`, `Seurat` and `biclust`.

Now we can give a simple explanation on how these module work

- `simulation.R`

  This file includes the generating functions of the simulated data. Specifically, `simu_mu(L,p,n)` is used to generate the parameter matrix $\pmb\mu$. It is computed by a multiplicative model $\pmb\mu = \pmb{WZ}$ where  **W**  is a *p*×*L* matrix and **Z** is a *L*×*n* matrix. The number of non-zero elements in each column of  **W**  is set as 50 and the number of non-zero elements in each row of  **Z** is randomly drawn from Poisson distribution with parameter 30. The row numbers with non-zero elements in **W** and **Z** are randomly drawn from 1 to p and 1 to *n*. And the elements of different columns of **W** are allowed to have overlaps. The non-zero elements of both **W** and **Z** are generated from normal distribution with mean 1.5 and standard deviation 0.1, and are randomly assigned to be positive or negative. The returned value is a list containing the matrix $W,Z,\mu$ and the true result $S$, which is a list including the groundtruth to be used to evaluate the performance of each method. a randomly sampled prior information is also included in the returned result. 

  ---

  `simu_nb_x(mu)` is used to generate the observed data matrix $\pmb X$ , the observed *p* × *n* data matrix **X** is generated by $\pmb{X=\mu+\epsilon}$ . The noise elements $\epsilon_{ij}$ are randomly chosen from $\mathcal{N}(0,4)$. For our Negtive Binomial case, each element of **X** is generated from $NB(r_j,\frac1{1+e^{-\mu_{ij}}})$ and the parameter $r_j$ is randomly sampled from 5 to 20

  ---

  `simu_noise_batch(X)` is used to simulate the batch noise and dropout on the simulated data matrix X. Here we separate the data matrix into three equal subsets, then apply different steady noise on them, which is seemed as the batch effects.

- `permutation.R`

  This file is actually a applets I write to calculate the permutaion of a given number. You can call it directly and get a matrix named `full_arrange`, which contains all the permutation results at each row. This can be used for matching the optimal bicluster in the evaluation stage. Of course you can modify it or write your own function to get a permutation result given a specific number.

- `match.R`

  This file is used to calculate the criterion we used in the paper: cluster error(CE, actually we calculate 1-CE), sensitivity and specificity. The CE value (actually 1-CE) is verisimilar to the IoU criterion in the computer vision area. So I write a function `calcu_supergrid()` to calculate the number of grid of the union of the biclustering result and the groundtruth. The flowchart is also embodied in the file  `analyse.R`.

- `GBC_EM.R`

- `DWL.R`

- `BIC_tuning.R`

  These 3 files are provided by [ZIYI et al](https://github.com/ziyili20/GBC). It contains the calculating process of GBC and sGBC and also used by our framwork. For more detail, you can refer to the [original paper](http://biostatistics.oxfordjournals.org).

- `plaid_Xmotif.R`

  This file is used to calculate the results of plaid and Xmotif. The input is only the data matrix and the expected number of biclusters. Make sure you have installed the R package `biclust` before using it. All the parameters of these two methods are set to default. You can modify it freely. 

- `analyse.R`

  This is the flowchart of the overall simulation study. You can change some parameters or do some repeats to get the s.d. of the simulation results.
